# E.M.S.U (Educational Management System United) - Technical Documentation

## Project Overview
A comprehensive, 3D interactive educational management platform designed specifically for Nigerian schools, combining traditional school administration with social networking features and advanced analytics.

## Architecture & Implementation

### Backend Implementation (Flask)
**File:** `backend/app.py`

#### Core Components:
1. **Database Architecture**
   - SQLite database with 10 core tables
   - Role-based user management (Proprietor, Principal, Teacher, Student, Parent)
   - School registration and multi-school support
   - Academic management (classes, subjects, grades)
   - Attendance tracking with automated notifications
   - Messaging system for internal communication
   - Social features (posts, interactions)
   - Financial management (fees, payments)

2. **Authentication System**
   - JWT-based authentication
   - Token-based authorization with role checking
   - Secure password hashing using SHA-256
   - Session management with automatic token refresh

3. **API Endpoints**
   - **Authentication:** `/api/login`, `/api/register`
   - **Schools:** CRUD operations for school management
   - **Students:** Student enrollment and management
   - **Grades:** Grade entry and report generation
   - **Attendance:** Daily attendance marking and tracking
   - **Messages:** Internal messaging system
   - **Posts:** Social networking features
   - **Analytics:** Dashboard analytics and reports

#### Key Features:
- Multi-school proprietor support
- Real-time messaging
- Grade analytics with performance insights
- Attendance rate calculations
- Social feed for educational collaboration
- Role-specific data access control

1. **Spacing Scale:** 8-step scale from 4px to 128px

4. **Shadow System:** 4-tier elevation system
5. **Animation Timing:** 3 speed variants (0.15s, 0.25s, 0.4s)

1. **Backend:**
   ```bash
   cd backend
   python app.py
   ```

2. **Frontend:**
   ```bash
   npm run start
   ```

## API Documentation

### Authentication Endpoints:
- `POST /api/login` - User authentication
- `POST /api/register` - New user registration

### Core Endpoints:
- `GET /api/schools` - List all schools
- `POST /api/schools` - Create new school
- `GET /api/students` - List students
- `POST /api/grades` - Add student grade
- `GET /api/grades/student/{id}` - Get student grades
- `POST /api/attendance` - Mark attendance
- `POST /api/messages` - Send message
- `GET /api/messages` - Get user messages
- `POST /api/posts` - Create social post
- `GET /api/posts` - Get social feed
- `GET /api/analytics/dashboard` - Get dashboard data
